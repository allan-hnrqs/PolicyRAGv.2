\begin{frame}{Problem}
  \begin{itemize}
    \item Procurement-policy QA is a high-exactness RAG problem.
    \item The system must retrieve relevant evidence and avoid inventing missing identifiers, forms, contacts, or workflow steps.
    \item Evaluation has to check both coverage and faithfulness.
  \end{itemize}
\end{frame}

\begin{frame}{Pipeline Overview}
  \begin{enumerate}
    \item Collect source pages
    \item Normalize and enrich documents
    \item Chunk them
    \item Build the hybrid retrieval index
    \item Retrieve and pack evidence
    \item Generate the answer
    \item Evaluate with multiple measurement lanes
  \end{enumerate}
  \vspace{0.5em}
  Current system: Buyer’s Guide-first hybrid RAG with query decomposition, strict
  evaluation, and a narrow exactness-specific post-draft correction path.
\end{frame}

\begin{frame}{Key Terms}
  \begin{itemize}
    \item \textbf{Canonical parity:} the umbrella benchmark family; in practice this means the \texttt{parity19\_dev} and \texttt{parity19\_holdout} splits together.
    \item \textbf{\texttt{parity19\_dev}:} the 9-case development split used during tuning.
    \item \textbf{\texttt{parity19\_holdout}:} the 10-case protected split used during promotion checks.
    \item \textbf{Tuning:} trying ideas, comparing variants, and adjusting methods against the dev split.
    \item \textbf{Promotion:} deciding whether a method deserves to move onto \texttt{main} or become a stronger default after holdout checks.
    \item \textbf{p95 / p99:} the 95th / 99th percentile. Example: p99 chunk size means 99\% of chunks are at or below that length.
    \item \textbf{Sliding-window chunking:} fixed-size text windows, usually with overlap, instead of structure-aware paragraph/section blocks.
    \item \textbf{Intervention-only evaluation:} baseline stays unchanged unless the conditional method actually fires.
  \end{itemize}
\end{frame}

\begin{frame}{How to Read the Metrics}
  \begin{itemize}
    \item \textbf{Recall score in the charts:} mean required-claim recall over the benchmark surface
    \item For each case:
      \begin{itemize}
        \item the judge checks each authored required claim
        \item case recall = supported required claims / total required claims
      \end{itemize}
    \item Example: a score of 0.8889 means about 88.89\% of required claims were covered on average
    \item \textbf{Pairwise:} blind A/B comparison between two full answers
      \begin{itemize}
        \item the judge picks control, candidate, or tie
        \item criteria: coverage, faithfulness, forbidden/safety handling, and abstention appropriateness
        \item example: 5-4-0 means control won 5, candidate won 4, ties 0
      \end{itemize}
    \item We did not replace recall with pairwise: recall stays the main scalar, while pairwise is a secondary promotion check because scalar recall alone sometimes rewards longer but less faithful answers
    \item We also track forbidden-claim violations and abstention accuracy separately
  \end{itemize}
\end{frame}

\begin{frame}{Chunking in Research Context}
  \begin{itemize}
    \item External guidance is directional, not absolute:
      \begin{itemize}
        \item Cohere recommends short snippets, around $\pm$400 words
        \item Databricks suggests starting tests around 256 / 512 / 1024 tokens
        \item Recent legal RAG work favors clause-boundary segmentation over giant sections
      \end{itemize}
    \item Our sliding-window test used 1200-char windows with 200-char overlap
    \item Under the repo's token heuristic, that is roughly a 300-token window with 50-token overlap
    \item So the size was plausible in the abstract, but not a clean legal-style test here:
      \begin{itemize}
        \item it removed section and heading structure
        \item it produced only 1872 chunks versus 5979 section chunks
      \end{itemize}
  \end{itemize}
\end{frame}

\begin{frame}{How Query Decomposition Works}
  \begin{itemize}
    \item A Cohere planner first breaks one buyer question into a small set of focused retrieval queries
    \item In the baseline profile, the planner is allowed to add at most 2 extra queries
    \item The planner is instructed to:
      \begin{itemize}
        \item target distinct aspects of the original question
        \item preserve procurement terms and source-specific language
        \item return JSON only
      \end{itemize}
    \item The retriever then:
      \begin{itemize}
        \item runs hybrid retrieval for the original query plus the expanded queries
        \item takes up to 24 candidates per query
        \item fuses them with reciprocal rank fusion
        \item reranks, then packs the final evidence bundle
      \end{itemize}
    \item This is why the gain is not just “more words”; it is a more targeted retrieval plan
  \end{itemize}
\end{frame}

\begin{frame}{Evaluation and Promotion}
  \begin{itemize}
    \item Primary lane:
      \begin{itemize}
        \item deterministic retrieval metrics
        \item structured Cohere judge
        \item required-claim recall
        \item forbidden-claim violations
        \item abstention accuracy
      \end{itemize}
    \item Secondary lanes:
      \begin{itemize}
        \item Ragas
        \item OpenAI pairwise A/B
      \end{itemize}
    \item Pairwise judge criteria:
      \begin{itemize}
        \item cover more required claims
        \item stay faithful and avoid unsupported details
        \item avoid forbidden or unsafe content
        \item abstain appropriately when the case expects abstention
      \end{itemize}
    \item Conditional methods are judged with intervention-only composite evaluation.
  \end{itemize}

\begin{frame}{Worked Example: Query Decomposition Helped Retrieval}
  \small
  \textbf{Case:} \texttt{HR\_008}

  \vspace{0.4em}
  \textbf{Prompt:}
  \begin{quote}
    What is the actual contractual difference between a standing offer and a
    supply arrangement? When does Canada become legally bound under each one?
  \end{quote}

  \begin{columns}[T]
    \begin{column}{0.48\textwidth}
      \textbf{Baseline retrieval}
      \begin{itemize}
        \item Top evidence was dominated by \emph{Using a standing offer}
        \item Example chunk:
      \end{itemize}
      \scriptsize
      ``A standing offer is not a contract. A contract is established with each
      call-up for goods or services made against the standing offer.''
      \normalsize

      \vspace{0.3em}
      \textbf{Result:} recall 0.3333
    \end{column}
    \begin{column}{0.48\textwidth}
      \textbf{With query decomposition}
      \begin{itemize}
        \item Added targeted retrieval queries about standing offers vs supply arrangements
        \item Newly packed chunk:
      \end{itemize}
      \scriptsize
      ``There is no contractual obligation between Canada and the offeror to buy
      goods or services at the supply arrangement stage. There is a contractual
      obligation with the award of each contract.''
      \normalsize

      \vspace{0.3em}
      \textbf{Result:} recall 1.0
    \end{column}
  \end{columns}
\end{frame}

\begin{frame}{Worked Example: Exactness Path Prevented an Unsupported Answer}
  \small
  \textbf{Case:} \texttt{HR\_037}

  \vspace{0.4em}
  \textbf{Prompt:}
  \begin{quote}
    What exact form number or file name do I need for the specimen signature
    card template in the GC network folder when approvals go beyond delegated
    authority?
  \end{quote}

  \textbf{Retrieved evidence excerpt:}
  \scriptsize
  ``The Delegation of Authorities instrument and Schedule 3 specimen signature
  card templates for the Acquisitions Program can be found in the Delegation of
  Authorities ... folder.''

  \vspace{0.3em}
  ``PSPC supervisors ... complete and sign a Schedule 3 Specimen Signature Card
  Template ... This form documents the scope of the incumbent's delegated
  approval and signing authority.''
  \normalsize

  \vspace{0.1em}
  \begin{columns}[T]
    \begin{column}{0.48\textwidth}
      \textbf{Baseline answer}
      \begin{itemize}
        \item Gave a specific file/form name as if it were established
        \item Result:
      \end{itemize}
      \scriptsize
      ``The exact form number or file name ... is the Schedule 3 Specimen
      Signature Card Template.''
      \normalsize

      \vspace{0.3em}
      Recall 0.3333, forbidden 1
    \end{column}
    \begin{column}{0.48\textwidth}
      \textbf{Exactness path answer}
      \begin{itemize}
        \item Explicitly said the evidence does not provide the exact file name
        \item Result:
      \end{itemize}
      \scriptsize
      ``The exact form number or file name ... is not provided in the evidence.''
      \normalsize

      \vspace{0.3em}
      Recall 0.6667, forbidden 0
    \end{column}
  \end{columns}
\end{frame}

\begin{frame}{Worked Example: Why Broad Mode-Aware Answering Was Rejected}
  \small
  \textbf{Case:} \texttt{HR\_001}

  \vspace{0.4em}
  \textbf{Prompt:}
  \begin{quote}
    We received a bid after the deadline and the supplier says they sent it on
    time. What proof counts, and what are we supposed to do with the offer?
  \end{quote}

  \begin{columns}[T]
    \begin{column}{0.48\textwidth}
      \textbf{Query-decomposition baseline}
      \begin{itemize}
        \item Covered acceptable proof
        \item Covered handling of late offers
        \item Covered offeror responsibility and return handling
      \end{itemize}

      \vspace{0.3em}
      \textbf{Result:} recall 1.0
    \end{column}
    \begin{column}{0.48\textwidth}
      \textbf{Mode-aware branch}
      \begin{itemize}
        \item Still covered acceptable proof
        \item But dropped part of the handling logic
        \item Judge notes: missing support for offeror responsibility and return based on delivery method
      \end{itemize}

      \vspace{0.3em}
      \textbf{Result:} recall 0.5
    \end{column}
  \end{columns}

  \vspace{0.5em}
  This is why rebuilt-39 gains alone were not enough for promotion.
\end{frame}
