\begin{frame}{Problem}
  \begin{itemize}
    \item Procurement-policy QA is a high-exactness RAG problem.
    \item The system must retrieve relevant evidence and avoid inventing missing identifiers, forms, contacts, or workflow steps.
    \item Evaluation has to check both coverage and faithfulness.
  \end{itemize}
\end{frame}

\begin{frame}{Pipeline Overview}
  \begin{enumerate}
    \item Collect source pages
    \item Normalize and enrich documents
    \item Chunk them
    \item Build the hybrid retrieval index
    \item Retrieve and pack evidence
    \item Generate the answer
    \item Evaluate with multiple measurement lanes
  \end{enumerate}
  \vspace{0.5em}
  Current system: Buyer’s Guide-first hybrid RAG with query decomposition, strict
  evaluation, and a narrow exactness-specific post-draft correction path.
\end{frame}

\begin{frame}{Source Design}
  \begin{itemize}
    \item Buyer’s Guide is the primary retrieval surface.
    \item Buy Canadian policy and the TBS directive are supporting sources.
    \item Collection, normalization, chunking, retrieval, answering, and evaluation are separate subsystems.

\begin{frame}{Evaluation and Promotion}
  \begin{itemize}
    \item Primary lane:
      \begin{itemize}
        \item deterministic retrieval metrics
        \item structured Cohere judge
        \item required-claim recall
        \item forbidden-claim violations
        \item abstention accuracy
      \end{itemize}
    \item Secondary lanes:
      \begin{itemize}
        \item Ragas
        \item OpenAI pairwise A/B
      \end{itemize}
    \item Conditional methods are judged with intervention-only composite evaluation.
  \end{itemize}
\end{frame}
